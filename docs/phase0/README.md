# フェーズ0 WASM技術検証

このディレクトリは、正本計画「フェーズ0」の再現可能な検証記録を置く場所です。追加機能・ライセンス対応後の同梱候補は次の構成に固定します。

```text
画像
  → OpenMVG (MPL-2.0) によるSfM
  → 自前MVS (本リポジトリと同じMIT) による密点群
  → PoissonRecon (MIT)
  → Manifold (Apache-2.0)
  → fTetWild (MPL-2.0)
  → 外部FEMソルバ向け出力
```

OpenMVS、TetGen、CGAL Shape Detection、Gmshは、MITで公開・配布するアプリへの同梱候補にしません。libiglを評価する場合はMPL-2.0のコアだけを対象とし、`igl/copyleft/` は利用しません。

## 検証の始め方

```bash
./scripts/check-phase0-toolchain.sh
cd app
npm ci
npm run build
```

環境診断はツールの有無を表示するだけで、インストール、clone、既存ファイルの変更は行いません。Emscripten SDKを導入した検証環境では、検証対象のリビジョン、ビルドオプション、パッチ、入力データのハッシュ、端末、ブラウザ、処理時間、ピークメモリ、出力品質を日付入りのMarkdownへ記録します。

## 検証順序と判定

| 項目 | 最小入力 | 記録する値 | 次へ進む条件 |
|---|---|---|---|
| 0-3 OpenMVG WASM | 20枚の縮小画像 | ビルド可否、処理時間、登録カメラ数、再投影誤差、ピークメモリ | ブラウザWorker内で完走し、成果物をIndexedDBへ保存できる |
| 0-4 自前MVS | 0-3の疎点群とカメラ | 点数、被覆率、既知寸法誤差、処理時間、ピークメモリ | L型ブラケットの主要面と穴周辺を後段処理に渡せる |
| 0-5 Poisson→Manifold→fTetWild | 密点群 | 水密性、自己交差、頂点/面/四面体数、負体積、アスペクト比 | 外部ソルバへ渡せる正体積四面体メッシュを生成できる |
| 0-6 2点スケール | 合成デモとCAD既知寸法 | 入力距離、倍率、再計測誤差、PLY/STL座標 | 元段階データを保持したまま表示・出力座標へ倍率が一貫して反映される |
| 0-7 端末性能 | Android中位機、PC | ブラウザ/OS、時間、メモリ、入力上限 | 端末別の画像枚数・解像度上限を明文化できる |
| 0-8 CAD比較 | 正解CAD | 幅、高さ、奥行き、穴径、体積の誤差 | 到達精度と対象外条件をREADME/使用書へ明記できる |

長時間ループをアプリへ載せる際は、`app/src/jobs/runner.ts` のジョブとして実装し、Worker内の反復単位ごとに `saveCheckpoint()` と `throwIfStopped()` 相当の境界を設けます。成果物は既存の `reconstructStub.ts` の契約に合わせ、段階データを上書きせず新しい `seq` として追記します。
